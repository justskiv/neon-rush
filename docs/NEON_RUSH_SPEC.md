# NEON RUSH — Спецификация игры

## Обзор

**Жанр:** Аркадный вертикальный раннер-гонка (вид сверху, скроллинг вниз→вверх)
**Движок:** Ebitengine (Go)
**Стиль:** Неоновый ретро, минимализм, процедурная графика
**Вдохновение:** Road Fighter (NES), Spy Hunter, Enduro (Atari)

Игрок управляет машиной, мчащейся по бесконечному шоссе. Уклоняется от трафика, собирает бонусы, набирает очки. Скорость постоянно растёт. Главный кайф — near-miss система (чем ближе проехал к другой машине, тем больше очков) и комбо-цепочки.

## Геймплей: Детальное описание

### Управление
- **←/→ (A/D):** Перемещение влево-вправо (плавное, не по полосам)
- **↑ (W):** Ускорение (временный буст скорости +30%)
- **↓ (S):** Торможение (скорость -50%, машины-NPC летят быстрее относительно игрока)
- **Space:** Нитро-буст (если есть заряд; 3 секунды неуязвимости + двойная скорость)
- **Esc:** Пауза
- **Enter:** Подтверждение в меню

### Физика движения игрока
```
// Боковое перемещение: плавное с инерцией
lateralVelocity += input * acceleration
lateralVelocity *= friction (0.85)
player.X += lateralVelocity

// Ограничение: игрок не может выехать за обочину
// При выезде на обочину — замедление + тряска
```

### Система Near-Miss (ключевая механика!)
Когда NPC-машина проходит мимо игрока:
1. Вычислить минимальное расстояние между бортами машин по X
2. Если `distance < NearMissThreshold` и **не было столкновения**:
   - Начислить `NearMissBonus * comboMultiplier` очков
   - Увеличить `comboMultiplier` на 1 (до макс.)
   - Показать "+50 NEAR MISS!" текст, плывущий вверх
   - Вспышка неона на машине игрока
   - Звук "woosh"
3. Таймер комбо сбрасывается при каждом near-miss
4. Если таймер истёк — комбо сбрасывается до x1

### Система столкновений
- **Столкновение с NPC:** Потеря жизни (или Game Over в хардкоре). Экран трясётся, искры, взрыв. NPC разлетается. Скорость падает до минимума. Комбо сбрасывается.
- **Столкновение с обочиной:** Замедление, искры от трения, небольшая потеря управления.

### Система очков
```
Базовые очки:       1 за каждый тик (за "проезд")
Near-miss:          50 × comboMultiplier
Обгон NPC:          10 (NPC ушёл за нижний край экрана)
Подбор топлива:     25
Подбор нитро:       15
Зона скорости:      очки ×2 пока в зоне

Итого = сумма за сессию → таблица лидеров (локальная)
```

### Прогрессия сложности
Сложность растёт плавно и непрерывно:
```
Время 0–15с:    Разминка. Мало машин, медленная скорость.
Время 15–60с:   Нарастание. Больше NPC, скорость растёт.
Время 60–120с:  Средне. NPC начинают менять полосы.
Время 120–180с: Сложно. Встречные машины (едут навстречу!).
Время 180с+:    Хаос. Максимальная плотность, все виды NPC.
```

### Зоны/Биомы (смена окружения)
Каждые ~30 секунд плавный переход цветовой палитры:
1. **Ночной город** (синий/фиолетовый неон)
2. **Пустынное шоссе** (оранжевый/жёлтый)
3. **Дождь** (серый/голубой, полоски дождя поверх)
4. **Неоновый туннель** (розовый/зелёный, стены по бокам)
5. **Закат** (красный/оранжевый градиент)

### Общий стиль
- **Палитра:** Неоновые цвета на тёмном фоне. Свечение (glow) через полупрозрачные слои.
- **Всё рисуется программно** — никаких внешних файлов изображений.
- **Пиксельный масштаб:** Логический экран 400×600, окно 800×1200 (×2 масштаб).

```

Тип 4: "Police" — 28×48px, чёрно-белый + мигалка (красно-синий)
  Появляется после 120с. Активно преследует игрока — подруливает к его полосе.

Тип 5: "Oncoming" — 26×44px, с включёнными фарами (два жёлтых круга спереди)
  Встречная машина! Едет НАВСТРЕЧУ (сверху вниз в 2× скорости). Появляется после 180с.
```

### Дорога
```
Фон:           Тёмно-серый (#1A1A2E)
Асфальт:       Серый (#2D2D44) прямоугольник от RoadLeft до RoadRight

- Левая сторона: Силуэты зданий (прямоугольники разной высоты, тёмные + светящиеся окна)
- Правая сторона: Аналогично
- Фонарные столбы: Вертикальные линии + точка-свет наверху, проносятся мимо
```

### Частицы и эффекты
```
1. Следы от колёс:     Две полупрозрачные серые линии за машиной при торможении
2. Искры столкновения: 8–12 жёлтых пикселей, разлетающихся от точки удара
3. Near-miss вспышка:  Кратковременное увеличение яркости контура машины
4. Нитро-пламя:        Оранжево-жёлтые "языки" за машиной (3–4 прямоугольника, мерцают)
5. Дождь (зона 3):     Тонкие полупрозрачные белые линии (10×2px) летящие вниз
6. Взрыв при смерти:   Расширяющийся круг белого цвета + разлетающиеся прямоугольники (осколки)
7. Комбо-текст:        "+50 ×3 NEAR MISS!" — белый текст, поднимается вверх и затухает
```

## Дизайн-спецификация: Звук

### Подход: процедурная генерация PCM

Все звуки генерируются кодом как `[]byte` (PCM signed 16-bit LE, stereo, 48000Hz) и заворачиваются в `audio.Player`.

### Звуки

```
1. ENGINE_LOOP (непрерывный):
   - Сумма двух пилообразных волн (sawtooth): 80 Hz + 120 Hz
   - Частота модулируется текущей скоростью (80–200 Hz)
   - Низкая громкость (фоновый гул)
   - Реализация: io.Reader, который генерирует сэмплы на лету

2. NEAR_MISS_WOOSH (0.2с):
   - Белый шум с быстрым fade-in/fade-out
   - Доплер-эффект: pitch понижается за 0.2 секунды
   - Генерировать при инициализации, хранить []byte

3. CRASH_BANG (0.5с):
   - Белый шум с резким fade-out
   - Наложение низкочастотного удара (60 Hz sine, 0.1с)

4. PICKUP_BLING (0.15с):
   - Два тона подряд: C5 (523 Hz) → E5 (659 Hz), каждый 0.07с
   - Sine wave, быстрый fade-out

5. NITRO_BOOST (1.0с):
   - Нарастающий белый шум + sine sweep от 200 Hz до 2000 Hz
   - Длится пока активен нитро, потом fade-out

6. COMBO_UP (0.1с):
   - Короткий восходящий чирп: sine sweep 400→800 Hz за 0.1с

7. MUSIC_BG (бесконечный loop):
   - Простой арпеджио-синтезатор:
   - Паттерн: 4 ноты по кругу (Am: A3→C4→E4→A4), каждая 0.2с
   - Square wave, тихий
   - Реализация: io.Reader, генерирующий паттерн в цикле
   - Или: вообще без музыки, только мотор + эффекты (может быть атмосфернее)
```

### Реализация аудио

```go
// Пример генерации звука
func generateSineWave(freq float64, duration float64, sampleRate int) []byte {
    numSamples := int(duration * float64(sampleRate))
    buf := make([]byte, numSamples*4) // 16-bit stereo = 4 bytes/sample
    for i := 0; i < numSamples; i++ {
        t := float64(i) / float64(sampleRate)
        sample := math.Sin(2 * math.Pi * freq * t)
        val := int16(sample * 0.3 * 32767) // 30% volume
        // Little-endian, stereo (same for both channels)
        buf[i*4] = byte(val)
        buf[i*4+1] = byte(val >> 8)
        buf[i*4+2] = byte(val)
        buf[i*4+3] = byte(val >> 8)
    }
    return buf
}

// Для движка: io.Reader, который генерирует PCM на лету
```

### ФАЗА 1: Ядро (Playable Prototype)
**Цель:** Машина едет, дорога скроллится, трафик появляется, столкновения работают, очки считаются.

**Что делать:**
1. Создать Go-модуль, подключить `github.com/hajimehoshi/ebiten/v2`
2. Реализовать `Game` struct с `Update()`, `Draw()`, `Layout()`
3. **Дорога (`road.go`):**
   - Рисовать серый прямоугольник (асфальт)
   - Рисовать обочины (жёлтые полосы по краям)
   - Рисовать штриховую разметку (3 линии между 4 полосами)
   - Разметка скроллится вниз с `scrollSpeed`
4. **Игрок (`player.go`):**
   - Рисовать машину как набор прямоугольников (см. спеку выше)
   - Управление ←→ с инерцией
   - Ограничение по X (не выезжать за дорогу)
5. **Трафик (`traffic.go`):**
   - Спавн NPC-машин сверху на случайной полосе
   - NPC двигаются вниз (скорость = scrollSpeed × 0.7, т.е. медленнее фона)
   - Удаление NPC за пределами экрана
6. **Столкновения (`collision.go`):**
   - AABB-коллизия между игроком и NPC
   - При столкновении: Game Over
7. **HUD (`hud.go`):**
   - Показать очки (инкрементируются каждый тик)
   - Показать "скорость" (визуальное число из scrollSpeed)
8. **Game Over экран:**
   - Показать финальный счёт
   - Enter → рестарт
9. **Нарастание сложности:**
   - scrollSpeed медленно увеличивается
   - Интервал спавна NPC уменьшается

**Результат фазы:** Можно играть! Машинка едет, уклоняешься от трафика, набираешь очки, при столкновении — Game Over с показом счёта. Уже есть ощущение скорости и напряжения.

**Критерий готовности:** Игрок может проехать 30+ секунд при умелой игре.

### ФАЗА 4: Враги и хаос (Challenge & Depth)
**Цель:** Police, Oncoming, нефтяные пятна, монеты — глубина геймплея.

**Что делать:**
1. **Police NPC:**
   - Появляется после 120с
   - Активно подруливает к полосе игрока
   - AI: каждые 30 тиков смещается на 1px к X игрока
   - При "поимке" (столкновении) — Game Over с особым текстом "BUSTED!"
2. **Oncoming NPC:**
   - Появляется после 180с
   - Летит сверху вниз с удвоенной скоростью (навстречу!)
   - Фары: два жёлтых круга спереди (мигают)
   - Near-miss с Oncoming даёт ×2 бонус (опаснее → больше награда)
3. **Нефтяные пятна:**
   - Спавнятся на дороге после 90с
   - При наезде: lateralVelocity += random(-3, 3) на 60 тиков
   - Визуал: тёмное полупрозрачное пятно
4. **Монеты:**
   - Линии из 3–5 монет на одной полосе
   - Каждая = 100 очков
   - Собрать все в линии = бонус ×2
5. **Улучшить декор:**
   - Рекламные щиты по бокам (цветные прямоугольники с текстом)
   - Звёзды в ночном небе (мелкие белые точки, для зон 1 и 5)

**Результат фазы:** Разнообразный, сложный геймплей. Police создаёт напряжение, Oncoming — адреналин, масло — неожиданность. Монеты мотивируют рисковать ради маршрута.

**Критерий готовности:** Игрок после 3 минут чувствует, что сложность честная, но жёсткая.

## Технические заметки для агента

### Зависимости
```
go get github.com/hajimehoshi/ebiten/v2
```
Это единственная внешняя зависимость. Всё остальное — стандартная библиотека Go.

### Рисование

Ebitengine рисует через `*ebiten.Image`. Основные операции:
- `screen.Fill(color)` — залить цветом
- `screen.DrawImage(img, opts)` — нарисовать изображение
- Для прямоугольников: создать `ebiten.NewImage(w,h)`, залить цветом, рисовать с opts

Для glow-эффектов: рисовать тот же прямоугольник бо́льшего размера, полупрозрачным, под основным.

```go
// Пример: нарисовать прямоугольник с glow
func drawGlowRect(screen *ebiten.Image, x, y, w, h float64, clr color.RGBA, glowSize float64) {
    // Glow layer (увеличенный, полупрозрачный)
    glow := ebiten.NewImage(int(w+glowSize*2), int(h+glowSize*2))
    glowClr := color.RGBA{clr.R, clr.G, clr.B, 40}
    glow.Fill(glowClr)
    op := &ebiten.DrawImageOptions{}
    op.GeoM.Translate(x-glowSize, y-glowSize)
    screen.DrawImage(glow, op)
    
    // Main rect
    rect := ebiten.NewImage(int(w), int(h))
    rect.Fill(clr)
    op2 := &ebiten.DrawImageOptions{}
    op2.GeoM.Translate(x, y)
    screen.DrawImage(rect, op2)
}
```

**ВАЖНО:** Не создавать `ebiten.NewImage` каждый кадр! Создать один раз и переиспользовать. Иначе — утечка памяти и тормоза.

### Текст

Для MVP использовать `ebitenutil.DebugPrintAt(screen, text, x, y)`.
Для финальной версии — `text/v2` с embedded шрифтом.

Можно embed шрифт через:
```go
import "golang.org/x/image/font/basicfont"
```
Или использовать встроенный моноширинный шрифт ebitengine.

### Аудио

Аудио-контекст создаётся один раз:
```go
audioContext := audio.NewContext(48000)
```

Для процедурного звука: реализовать `io.Reader`, который генерирует PCM на лету.

### Ввод

```go
if ebiten.IsKeyPressed(ebiten.KeyLeft) { ... }
if ebiten.IsKeyPressed(ebiten.KeyRight) { ... }
if inpututil.IsKeyJustPressed(ebiten.KeySpace) { ... }  // Для одиночных нажатий
```

### Производительность

- Переиспользовать `*ebiten.Image` объекты (создать в init, рисовать многократно)
- Частицы: пул объектов (slice с active/inactive)
- NPC: тоже пул (slice, переиспользовать при деспавне)
- Не аллоцировать в Update/Draw

### Структура Game

```go
type Game struct {
    // Состояние
    state      GameState // Menu, Playing, Paused, GameOver, Garage
    
    // Игрок
    player     Player
    
    // Мир

    traffic    []TrafficCar
    items      []Item
    particles  []Particle
    
    // Системы
    score      ScoreSystem

    zone       ZoneSystem
    
    // Ресурсы (создаются один раз)
    carImages  map[string]*ebiten.Image
    itemImages map[string]*ebiten.Image
    
    // Мета

    tickCount  int

    StateMenu GameState = iota
    StatePlaying
```

## Советы по реализации

1. **Начинай с простого прямоугольника.** Машина игрока в фазе 1 — просто цветной прямоугольник. Детализируй позже.

2. **Скроллинг — это иллюзия.** Игрок не двигается вверх. Всё остальное двигается вниз. `scrollOffset` увеличивается каждый тик.

3. **NPC-спавн:** Случайная полоса, случайный тип (по весам, зависящим от времени). Проверить, что не спавнится поверх другого NPC.

4. **Near-miss — после того как NPC прошёл мимо.** Проверять, когда `npc.Y > player.Y + player.Height` (NPC уже позади), и только тогда считать минимальную дистанцию.

5. **Не усложняй физику.** Это аркада. Управление = прямое перемещение с инерцией, не нужна никакая симуляция.

6. **Цветовой lerp для зон:**
```go
func lerpColor(a, b color.RGBA, t float64) color.RGBA {
    return color.RGBA{
        R: uint8(float64(a.R) + (float64(b.R)-float64(a.R))*t),
        G: uint8(float64(a.G) + (float64(b.G)-float64(a.G))*t),
        B: uint8(float64(a.B) + (float64(b.B)-float64(a.B))*t),
        A: 255,
    }
}
```

7. **Пул частиц:**
```go
type Particle struct {
    active bool
    x, y, vx, vy float64
    life int
    color color.RGBA
}

// Создать 200 частиц при инициализации
particles := make([]Particle, 200)

// Активировать частицу:
func spawnParticle(particles []Particle, ...) {
    for i := range particles {
        if !particles[i].active {
            particles[i] = Particle{active: true, ...}
            return
        }
    }
}
```
